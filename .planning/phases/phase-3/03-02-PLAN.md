   - `generateRecommendation(result: RiskCalculationResult): string`

2. **Pattern Rules** (from CONTEXT.md):
   - Single severe: severity=severe → 60% risk
   - Multiple moderate: 3+ moderate in 7 days → 50% risk
   - Recurring same: same description 3+ times → 70% risk
   - Escalating: severity increasing over time → 75% risk
   - Multiple severe: 2+ severe in 30 days → 85% risk

3. **High-risk alert creation**:
   - When riskScore >= 70, create Alert record
   - Include symptom summary, risk level, recommendation

Create server/src/controllers/symptom.controller.ts:

1. **Endpoints**:
   - GET /api/pets/:id/symptoms - List symptoms (paginated, by date desc)
   - GET /api/pets/:id/symptoms/:symptomId - Get single symptom
   - POST /api/pets/:id/symptoms - Create symptom + AI analysis (SYMP-01, SYMP-05)
   - PUT /api/pets/:id/symptoms/:symptomId - Update symptom (SYMP-03)
   - DELETE /api/pets/:id/symptoms/:symptomId - Delete symptom (SYMP-04)
   - GET /api/pets/:id/symptoms/analysis - Get AI risk assessment (SYMP-05)

2. **POST /api/pets/:id/symptoms flow**:
   - Validate input with zod (description 1-500 chars, severity enum, date)
   - Create symptom record in DB
   - Fetch all symptoms for pet (last 30 days)
   - Run SymptomAnalysisService.analyzeSymptoms()
   - If riskScore >= 70, create Alert record
   - Return symptom with analysis result

3. **GET /api/pets/:id/symptoms/analysis flow**:
   - Fetch last 30 days of symptoms
   - Run analysis
   - Return RiskCalculationResult with disclaimer

Create server/src/routes/symptom.routes.ts:

1. Router with all symptom endpoints
